# 二次采访追问设计

犟朋友项目的采访通常不止一轮。第一轮采集主体经历和产品逻辑，第二轮深挖细节和情感层次。

## 设计原则

- **去苦情化**：不逼人卖惨。不问"你最苦的时候是什么"，问具体画面和选择
- **追原话**：当对方提到某个人（家人/朋友/导师），追问"他原话怎么说的"——原话比转述有力量
- **留白空间**：敏感话题（如亲人去世）让对方决定给多少，不追问细节。脚本可以用留白处理
- **不放过的信号**："算了不说了""当时也没想那么多"——这种收回去的话后面往往有东西。可以轻声追一句"是不想说，还是没想好怎么说"

## 四层递进结构

### 第一层：补充空缺信息（轻松切入）
- 第一个客户、第一笔收入、第一个员工
- 培训/学习中的趣事（如"考第一日本人什么反应"）
- 作用：热身，不让对方上来就沉重

### 第二层：经历里的画面感（为脚本积累场景）
- 关键事件的具体时刻："你是什么时候开始觉得'我得做这个产品'？"
- 转变过程的细节："从药学转到市场，有没有人觉得可惜了？"
- 寂寞的具体画面："寂寞到什么程度？有没有具体的某一天？"
- 作用：从"我做了什么"进入"我怎么走过来的"

### 第三层：人物复杂性和矛盾点
- 不理解的人——最不理解你的人是谁，说过什么
- 犟有撞过南墙吗？犟错过吗？
- 失败过吗？失败了怎么面对？
- 作用：打破"犟=正确"的单一叙事，让人物立体

### 第四层：松弛面和情感底层
- 有没有讨厌但不得不做的事
- 被年轻人触动过的具体瞬间
- 有没有哭过？如果没有，有没有特别安静、不想说话的时刻
- 作用：在"犟"的坚硬外壳下找到柔软层

## 追问技巧

| 信号 | 追问 |
|------|------|
| "算了不说了" | 轻声："是不想说，还是没想好怎么说" |
| "当时也没想那么多" | "那什么时候开始想的" |
| 出现具体人名 | "他原话怎么说的" |
| 说到某个产品眼眶有变化 | 停。等他继续说。不要马上换话题 |
| "这个不能往上面写" | 尊重，不追问。但记下来——这是真实的边界 |
